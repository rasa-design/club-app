# ペルソナ定義

このアプリのターゲットユーザーを以下のペルソナとして定義する。
実装・UI設計・文言・優先度の判断は、常にこのペルソナを基準にすること。

---

## ペルソナ A：一般保護者（メインユーザー）

| 項目 | 内容 |
|------|------|
| 役割 | 中学校クラブに子どもが所属する保護者 |
| デバイス | スマートフォン（iOS / Android）がメイン。PCはほぼ使わない |
| ITリテラシー | 低〜中。LINEや地図アプリは使えるが、webアプリに慣れていない人も多い |
| 目的 | ① 大会・行事の日程を手軽に確認したい　② 月謝の支払い状況を確認したい |
| ゴール | アプリを開いてすぐ必要な情報にたどり着けること |
| 不安・懸念 | 操作が難しそう、どこを見ればいいかわからない、子どもの情報が外部に漏れないか |

### 実装指針
- モバイルファースト。タップしやすいボタンサイズ（最低 44px）
- 文字は大きめ・わかりやすい日本語。専門用語・英語UIは避ける
- ページ遷移を最小限に。トップから2タップ以内で目的の情報へ
- 余計な機能は見せない。閲覧に必要な操作だけシンプルに提供する
- エラー時のメッセージは日本語で具体的に

---

## ペルソナ B：会計係（管理者）

| 項目 | 内容 |
|------|------|
| 役割 | クラブの会計を担当する保護者（当番制・年1〜2名） |
| デバイス | スマートフォンがメイン。たまにPC |
| ITリテラシー | 中程度。表計算ソフトは使える。webアプリ管理経験はほぼなし |
| 目的 | 月謝の支払い済みを◯で記録したい。誰が未払いかすぐ把握したい |
| ゴール | 集金のたびに手間なく記録できること。操作ミスを起こしにくいこと |
| 不安・懸念 | 誤操作で記録が消えないか、パスワードを忘れたらどうするか |
| 引き継ぎ | 年度ごとに担当者が変わるため、初見でも使えることが重要 |

### 実装指針
- ログイン後のUIはシンプルに。管理機能が多すぎると混乱する
- ◯のトグルは明確なフィードバック（視覚的変化）を即時に返す
- 削除など取り消しの難しい操作には確認ダイアログを必ず出す
- パスワードは `.env.local` で管理し、変更手順を README に記載する
- 操作ログは不要（個人宅サーバーではなくシンプル運用を想定）

## ペルソナ C：部活顧問・管理担当教員（将来想定・現時点は対象外）

現バージョンでは対象外。将来的に追加が必要になった場合に検討する。

---

## 共通設計方針

- **プライバシー**: 氏名・学年など個人情報は最小限。金額は表示しない
- **可用性**: 特定のOSやブラウザに依存しない。Safari / Chrome で動作確認優先
- **シンプルさ優先**: 機能追加よりも、今ある機能が確実に動くことを重視する
- **日本語統一**: UI文言・コメント・変数名コメントはすべて日本語または文脈で明確なもの
